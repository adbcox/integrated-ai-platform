## 4. What Is the Training Pipeline?

**Answer: Real LoRA fine-tuning infrastructure — isolated, functional but untested at scale.**

- Location: `framework/model_trainer.py`, `bin/run_training_cycle.py`, `bin/collect_training_data.py`, `bin/plan_training_distillation.py`, `domains/training_viz.py`
- What it does:
  - Collects execution outcomes as training examples (`bin/collect_training_data.py`)
  - Fine-tunes `Qwen/Qwen2.5-Coder-7B-Instruct` (~14GB HuggingFace weights) using LoRA
  - Exports fine-tuned adapter as GGUF for loading into Ollama
  - Runs in isolated `~/training-env` venv (Python 3.12 + torch + peft + transformers)
  - **NOT** in system Python — explicitly isolated to avoid dep conflicts with aider
- Current status: Code is complete, environment is defined in `docs/TRAINING.md`, but requires:
  - Sufficient VRAM (Mac Studio with 96GB unified memory is the target)
  - HuggingFace weight download (~14GB one-time)
  - Execution outcome data to train on
- **VERDICT: Real, non-trivial pipeline. Designed for the Codex 5.1 replacement goal (fine-tune local model on platform's own execution patterns). Blocked by needing Mac Studio GPU access.**
