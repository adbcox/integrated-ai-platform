# Review Cadence

## Purpose

This document defines a practical review cadence for keeping the roadmap, architecture alignment, and execution readiness current.

## Suggested cadence

### On intake

Every new roadmap intake should update the canonical roadmap surfaces in the same intake cycle.

### Weekly or active-planning cadence

Review:

- active strategic cluster
- new items added since the last review
- queue-rank drift inside top priority classes
- newly blocked or newly ready items
- grouped execution opportunities
- external-system dependencies that need documentation packs

### Milestone or package-closeout cadence

Review:

- whether status sync reflects reality
- whether strategic ordering still matches architecture priorities
- whether grouped execution recommendations should change after completed work

### Architecture-change cadence

When a material architecture change occurs, update architecture and roadmap alignment docs in the same workstream.

## Rule

A roadmap that is not reviewed on a cadence becomes a historical artifact instead of a planning system.
