# Roadmap Item Maturity Model

## Purpose

This model explains how mature a roadmap item is from an intake/governance perspective, independent of status.

## Maturity levels

### M0 — Idea capture
The item exists as a concept but is not yet normalized enough for canonical planning.

### M1 — Normalized backlog item
The item has a stable ID, category, title, basic description, and enough structure to live in the canonical backlog.

### M2 — Prioritized planning item
The item has metrics, queue placement, dependency awareness, and architecture alignment.

### M3 — Execution-shaped item
The item has clear scope, file/system impact understanding, and a bounded first milestone.

### M4 — Execution-ready item
The item satisfies the execution-readiness checklist and can enter implementation without major scope ambiguity.

### M5 — Validated/closeout-ready item
Implementation is materially complete and the remaining work is validation or closeout synchronization.

## Rule

Status and maturity are related but not identical.
An item may be accepted but still low maturity if its structure is weak.
