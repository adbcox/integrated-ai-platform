# Risk and Issue Management

## Purpose

This document distinguishes risks from issues and explains how they should be handled in roadmap planning.

## Definitions

- **Risk** — something that may go wrong or reduce value if not managed
- **Issue** — something currently wrong, blocked, missing, or conflicting

## Rule

Do not hide planning risk inside vague prose.
Record major risk or issue posture explicitly where it materially affects sequencing or readiness.

## Typical roadmap risks

- architecture drift
- dependency uncertainty
- external-system uncertainty
- oversized grouped package scope
- missing documentation pack
- weak readiness despite apparent urgency

## Typical roadmap issues

- ID collision
- stale status surface
- unresolved blocker
- missing external docs
- conflicting authority surfaces
- mismatched strategic and operational ordering

## Escalation shorthand

If an issue changes status truth, priority ordering, or execution readiness, update the canonical roadmap surfaces in the same workstream.
