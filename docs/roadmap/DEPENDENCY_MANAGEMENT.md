# Dependency Management

## Purpose

This document explains how roadmap dependencies and blockers should be handled.

## Core rule

Dependencies should be explicit enough that pull planning is based on real sequencing, not guesswork.

## Dependency classes

- hard dependency
- soft dependency
- shared-surface dependency
- external-system dependency
- authority/documentation dependency

## Required handling

For meaningful items, record:

- what must exist first
- what can proceed in parallel
- what becomes simpler if grouped together
- what external system or documentation pack is required

## Blocker rule

If a roadmap item is blocked, the blocking condition should be explicit and should not be hidden in chat or implied by missing context.
