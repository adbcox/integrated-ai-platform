# Execution Readiness Checklist

## Purpose

Use this checklist before treating a roadmap item or grouped package as execution-ready.

## Checklist

- Is the item normalized in the canonical roadmap set?
- Is architecture alignment clear?
- Are dependencies and blockers identified?
- Are affected systems and expected file families identified?
- Is grouped-vs-separate execution considered where relevant?
- Are external documentation packs complete if external systems are involved?
- Is the first milestone bounded and coherent?
- Is there a clear status path into validation and completion?
- Are forbidden or read-only surfaces identified where needed?
- Is the item mature enough to avoid scope confusion during execution?

## Rule

Execution-ready should mean more than “someone wants this done.”
