## NEW finding — F1 extends to streaming mode

**Critical discovery during install verification.**

The original D-17-13 blocker was: **Ollama drops `tool_calls` in streaming
responses**. D-17-12 F1 verified that qwen3-coder:30b emits structured
`tool_calls` via Ollama — but only in `stream:false` mode (bench harness used
direct curl with stream disabled).

Goose 1.33.1's Ollama provider **hard-codes streaming on**:

```
"Ollama provider does not support non-streaming mode. All Ollama models
support streaming. Please remove 'supports_streaming: false' from your
provider configuration."
```

This means F1 from D-17-12 alone is INSUFFICIENT to unblock D-17-13 — we
need to verify the streaming case.

**Probe executed against Mac Studio Ollama 0.22.1, qwen3-coder:30b,
stream=true, tools=[get_weather]:**

```bash
curl -sN -X POST http://192.168.10.142:11434/api/chat -d '{
  "model": "qwen3-coder:30b",
  "stream": true,
  "messages":[{"role":"user","content":"What is the weather in Paris?
              Use the get_weather tool."}],
  "tools":[{...}]
}'
```

Response (first SSE chunk):

```json
{"model":"qwen3-coder:30b","created_at":"2026-05-03T21:14:26.570427Z",
 "message":{"role":"assistant","content":"",
   "tool_calls":[{"id":"call_uhvjy6re",
                  "function":{"index":0,"name":"get_weather",
                             "arguments":{"city":"Paris"}}}]},
 "done":false}
```

**Structured `tool_calls` array is populated in the streaming chunk.**

This is THE unblocker. The original local-tool-calling.md Finding 1 conclusion
("Ollama drops tool_calls from streaming responses") was correct at the time
for qwen2.5-coder family — but is no longer true for qwen3-coder family
running on Ollama 0.22.1.

Refines F1 chronicle:
- Finding 1 (qwen2.5-coder via Ollama 0.20.7 streaming): inline `<tools>` markup → no structured tool_calls. **Still true.**
- Finding 1.B (NEW): qwen3-coder:30b via Ollama 0.22.1 streaming: structured `tool_calls` emitted correctly. Resolves D-17-13 blocker.
- The dichotomy is along **(model-family × Ollama-version)** axis, not just one or the other.
