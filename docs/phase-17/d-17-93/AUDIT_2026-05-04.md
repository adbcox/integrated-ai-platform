## WP-04: Friction Analysis

### Finding F1 — No operator-facing CLI entrypoint (CRITICAL)

`domains/router.py` and `bin/aider_local.sh` are library/script primitives. There is
no single command an operator types to route a task through LOCAL_AIDER. The only
comparable surface is `claude-local` (Claude Code shell function) which is the
entrenched default.

The 98 router events were generated programmatically by a benchmark campaign — not
operator-initiated. Zero organic operator-driven LOCAL_AIDER invocations in the log.

**Fix:** `scripts/aider-task.sh <task-description> [file...]` — thin wrapper that calls
router classify, logs the decision, and executes `bin/aider_local.sh` or prints
escalation recommendation.

### Finding F2 — Timeout defaults too short (HIGH)

120s micro timeout and 180s default timeout caused 72/179 attempt failures (40%).
`qwen2.5-coder:14b` cold load on Mac Mini takes 20–40s; a 120s budget leaves only
80s for generation on non-trivial tasks. The 2026-04-15/16 campaign accumulated
failures that would deter any operator who observed them.

**Fix:** Raise micro timeout to 300s, default to 360s. Update `aider_local.sh`
defaults and the micro-mode constants in `aider_micro.sh`.

### Finding F3 — Model registry stale (MEDIUM)

`router.py` keyword path references `qwen2.5-coder:1.5b` (not installed on any host)
for the ultra-fast 1-file case. The router's model fallback list uses
`deepseek-coder-v2` as secondary. Current Mac Mini inventory: `qwen2.5-coder:14b`,
`qwen2.5-coder:7b`, `qwen2.5-coder:32b`, `devstral:latest`. `devstral` is not in
the router at all.

**Fix:** Update `MODELS` / model_order in `router.py` and `aider_local.sh` to match
current `ollama list` inventory. Remove `qwen2.5-coder:1.5b` references.

### Finding F4 — Task class never populated (MEDIUM)

All 98 router events have `task_class: null`. The learning layer accumulates history
by task_class, but callers never pass a class. The `task_class` axis in
`router_events.jsonl` is permanently null. Learning recommendations are made on a
single bucket ("coding") with no sub-class discrimination.

**Fix:** Add `--task-class` argument to the proposed `scripts/aider-task.sh` CLI so
operators can tag tasks. Align with C0/C1/C2/C3 taxonomy from D-17-90 prompt library.

### Finding F5 — Learning layer overrides keyword escalation (LOW)

When learning confidence ≥ 0.60, the keyword routing path (including the 6-file
escalation to CLAUDE_CODE) is bypassed entirely. This means a 10-file refactor task
would be routed to LOCAL_AIDER based on historical success rate, even if the keyword
rule explicitly escalates it. The learning layer winning over keywords is intentional
design, but operators should know the escalation keyword path is effectively disabled
whenever history exists.

**Not a bug** — document in `aider-task.sh` help output.

### Finding F6 — `bin/` was gitignored during early runs (INFORMATIONAL)

2026-04-15 aider.log shows: `Repo-map can't include /Users/admin/repos/integrated-ai-platform/bin — Has it been deleted from the file system but not from git?` This was an early platform state issue (bin/ was gitignored or missing), not a current problem. Resolved before 2026-04-16.

print route decision, exec `bin/aider_local.sh --message "$TASK" $FILES` if
