# Codex Control Mode

## Purpose

Use this mode when Codex is acting as a **control window**.
This mode is for truth, acceptance, closure review, contradiction detection, and bounded governance analysis.

## Best use cases

- completion verification
- blocker-chain review
- roadmap truth reconciliation
- contradiction detection across canonical/derived/summary surfaces
- acceptance audit after an execution pass
- post-pass readiness / closeout decisions

## Not for

- broad exploratory implementation
- open-ended feature ideation
- writing large amounts of new product code unless direct contradiction repair requires it

## Required reads

1. `docs/governance/CURRENT_OPERATING_CONTEXT.md`
2. `docs/roadmap/ROADMAP_AUTHORITY.md`
3. relevant canonical item YAML
4. relevant derived planning/dependency surfaces
5. relevant artifacts / validation outputs

## Operating rules

- be strict and evidence-first
- do not over-credit partial progress
- patch success is not item completion
- summary surfaces do not override canonical item YAML
- if closeout is only locally visible, do not accept it as final
- prefer identifying exact contradictions over narrative summaries

## Expected outputs

- acceptance decision
- exact files reviewed
- exact contradictions or confirmations found
- exact corrective actions if rejected
- explicit statement on whether completion/closure is justified

## Stop conditions

Stop when:
- the truth decision is clear and evidence-backed, or
- a direct contradiction requiring execution-mode repair has been identified
