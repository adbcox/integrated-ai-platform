## Section 5 — Migration / retirement plan

**Pattern:** PARK-RETIRE per established platform doctrine. **Out-of-repo
compose** — D#15 rewire-log entry required.

**Steps:**

1. **Stop the container:**
   ```bash
   docker stop homarr && docker rm homarr
   ```

2. **Park the compose block:** copy the homarr service block (and
   its three named-volume declarations) from
   `~/control-center-stack/stacks/ai-control/docker-compose.yml`
   into a new file
   `docker/_retired/homarr/docker-compose.parked.yml`
   (in-repo — the parked artifact lives in the repo even though the
   live compose was out-of-repo).

3. **Remove the live block from the out-of-repo compose:** edit
   `~/control-center-stack/stacks/ai-control/docker-compose.yml` to
   delete the homarr service block. Keep the three named-volume
   declarations IN PLACE (commented, with retirement date) so the
   volumes are not garbage-collected.

4. **D#15 rewire log:** create
   `docker/_rewire-log/2026-05-01-homarr-retirement.md` with:
   - Pre-snapshot of `~/control-center-stack/stacks/ai-control/docker-compose.yml`
     (full file).
   - Post-snapshot of the same file after homarr block removal.
   - Diff highlighting only the homarr-related deletions.
   - Reason: 17.F PARK-RETIRE.

5. **Preserve the named volumes:**
   `docker volume ls | grep -E "ai-control_homarr-(config|icons|data)"`
   — leave all three in place. Document names in the parked compose
   for future restoration.

6. **README:** add `docker/_retired/homarr/README.md` with:
   - Retirement date and rationale (zero traffic 7d).
   - Three volume names and the fact that the out-of-repo compose
     keeps them declared.
   - Restoration recipe.
   - Reference to this audit doc and the rewire-log entry.

**Restoration recipe (for the README):**

```bash
# 1. Restore the homarr block in the out-of-repo compose
#    (copy from docker/_retired/homarr/docker-compose.parked.yml)
$EDITOR ~/control-center-stack/stacks/ai-control/docker-compose.yml

# 2. Bring the container back
cd ~/control-center-stack/stacks/ai-control
docker compose up -d homarr

# 3. Volumes are auto-remounted (config, icons, data preserved)
```
