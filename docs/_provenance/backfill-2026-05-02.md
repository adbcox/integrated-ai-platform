## Method

Per Path A doctrine in `docs/architecture-facts/model-provenance.md`
(§"Backfill verdicts"), backfill verdicts use one of three sources,
named explicitly per row:

- **scan-confirmed:** `scripts/verify-model-provenance.sh` was run
  against the HF source repo; verdict + top-K captured in
  `docs/_provenance/<sanitized>.json`.
- **catalog-inferred:** T0.5 catalog enumeration confirmed the HF
  source's presence (or absence) in the deep-signals fingerprint DB;
  verdict deduced from catalog presence + variant-scoring caveat
  (fine-tunes match base at ~0.85–0.95).
- **mapping-inferred:** Ollama tag → HF source repo is deterministic
  per Ollama manifest; verdict inferred from the mapped HF repo's
  catalog status.

### T6 backfill correction

The original backfill (pre-T6 validation) predicted `nomic-embed-text`
at exit 1 unverified by reasoning "nomic family absent from catalog →
no match." The T6 scan-confirmed verdict is exit 2 marginal (score
0.653 against `deepseek-coder-7b-instruct-v1.5`, match_type=similarity).

**Why catalog-inference was insufficient here:** when the family is
absent, the kit can still surface weak similarity matches drawn from
unrelated families that happen to share architectural traits. These
fall in the [0.50, 0.70) marginal band, not below 0.50 unverified.
Catalog-inference reads "family present?" but the kit's actual scan
output reads "any match above 0.50?" — a strict superset that includes
cross-family similarity.

**Implication for future backfills.** When a family is absent from
the catalog, treat the catalog-inferred verdict as "exit 2 OR exit 1"
rather than committing to exit 1. Run the wrapper to disambiguate if
the verdict matters for an operational decision (e.g. retain vs replace).
For nomic-embed specifically, exit 2 marginal carries the same
operator-action implication as exit 1 unverified would have (gate
blocks, override required for forward pulls), so the operator-review
flagging below remains valid.

## Confirming an inferred verdict on demand

To upgrade any "inferred" row to "scan-confirmed":

```sh
scripts/verify-model-provenance.sh <hf-source-repo>
```

The wrapper writes the result to `docs/_provenance/<sanitized>.json`
and caches for 30 days. First run downloads the safetensors version
of the model from HuggingFace (sized per model, ~14–65 GB for the
inferred-row models above). Subsequent runs are cache hits.
