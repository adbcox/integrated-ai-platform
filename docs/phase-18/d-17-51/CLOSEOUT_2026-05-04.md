## MCP outliers — classification (the 3 FAILs)

The migration script reported 3 FAILs:
- `com.iap.mcp.docker`
- `com.iap.mcp.docs`
- `com.iap.mcp.filesystem`

**Root cause: stale duplicate-publisher plists.** The three plists are
pre-containerization residue. They invoke wrapper scripts at
`/Users/admin/mcp-servers/remote/mcp-{docker,docs,filesystem}.sh`
(2026-04-27) that call `npx supergateway` to wrap stdio MCP servers
into HTTP listeners on ports 8091/8092/8093.

Those ports are already owned, healthy and serving, by Docker
containers since 2026-05-03 02:54:
- `mcp-filesystem-remote` → `0.0.0.0:8091` (healthy)
- `mcp-docker-remote` → `0.0.0.0:8092` (healthy)
- `mcp-docs-remote` → `0.0.0.0:8093` (healthy)

Evidence chain:
1. Plists are well-formed and load (root:wheel, `RunAtLoad=true`,
   `KeepAlive=true`, `ThrottleInterval=10`).
2. Daemon logs (`/Users/admin/Library/Logs/iap/com.iap.mcp.*.out.log`)
   show the wrapper supergateway start cleanly — `[supergateway]
   Listening on port NNNN` — and then immediately receive `Caught
   SIGTERM. Exiting...`. Throttle/exit loop kicks in; launchctl gives
   up; `loaded=no` is reported by the verifier.
3. `lsof -iTCP:8091/8092/8093 -sTCP:LISTEN` shows the host-side
   listener is `ssh PID 4833 (/Users/admin/.colima/_lima/colima/ssh.sock
   [mux])` — Colima's host→lima-VM port forward exposing the container
   ports back to macOS, NOT a remote SSH session.
4. `curl /healthz` succeeds on all three (`8091:200 8092:200`;
   8093 returns the docs-MCP's own 404 on `/healthz` because that
   variant routes differently — service is up either way).
5. `mcp-docker-remote` has a compose file at
   `docker/mcp/docker-compose.mcp-docker-remote.yml`; the trio is the
   D#30 "permanently non-compose-hardened" set in CLAUDE.md (Obot /
   bare-docker-run lifecycle, not launchd lifecycle).

**Disposition: retire from launchd.** The canonical lifecycle for
these services is the Docker container substrate that already
hosts them. The launchd plists never got removed when the runtime
moved into Docker.

## Privileged retirement (operator-run, queued)

```bash
sudo launchctl bootout system/com.iap.mcp.docker || true
sudo launchctl bootout system/com.iap.mcp.docs || true
sudo launchctl bootout system/com.iap.mcp.filesystem || true
sudo rm /Library/LaunchDaemons/com.iap.mcp.docker.plist
sudo rm /Library/LaunchDaemons/com.iap.mcp.docs.plist
sudo rm /Library/LaunchDaemons/com.iap.mcp.filesystem.plist
```

Optional non-privileged cleanup (can be staged later, not gating):
move `/Users/admin/mcp-servers/remote/mcp-{docker,docs,filesystem}.sh`
into an `_archive/` subfolder with a README pointing to the
docker-compose path.
