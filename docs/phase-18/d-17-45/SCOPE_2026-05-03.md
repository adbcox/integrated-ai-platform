# D-17-45 — CMDB reconciliation scope (post-WP-04 decision)

**Date:** 2026-05-03
**Authority decision:** Option C (Hybrid boundary)
**Boundary lock:**
- NetBox: static asset/network/service catalog intent
- `~/.platform-registry/inventory.json`: runtime/container/route/dependency state
- `config/service-registry.yaml.DEPRECATED`: fallback-only until explicit deletion gate

## In-scope sub-deliverables

### SD-01 — Consumer-boundary enforcement
- Goal: every CMDB consumer reads the correct substrate for its question type.
- Scope:
  - Classify each consumer as `intent-layer` or `runtime-layer`.
  - Patch/confirm reader codepaths so they cannot silently cross layers.
  - Add explicit boundary notes to runbooks and architecture docs.
- Hard cap: **4h**
- Output:
  - consumer map (file + expected substrate)
  - code/config diffs for boundary routing
  - verification table (before/after)

### SD-02 — Schema identity unification contract
- Goal: establish shared identity contract across both canonical layers.
- Scope:
  - Define stable service identity tuple and alias policy.
  - Resolve known mismatches (`iap-dashboard` vs `ai-platform-dashboard`, similar pairs).
  - Define attribute ownership per field (who owns `port`, `host_mapped`, `depends_on`, etc.).
- Hard cap: **3h**
- Output: schema contract doc + compatibility matrix.

### SD-03 — Drift detection automation (hybrid-aware)
- Goal: automated detection of stranded/conflicting/stale states without mutating CMDBs.
- Scope:
  - Implement read-only drift job comparing NetBox intent vs runtime inventory.
  - Severity model: stale declarative, runtime-only, conflict, IPAM gap.
  - Emit machine-readable report + human markdown summary.
- Hard cap: **5h**
- Output: scheduled check + report artifacts + operator runbook.

### SD-04 — YAML fallback retirement gate
- Goal: safely retire `service-registry.yaml.DEPRECATED` only after objective gate pass.
- Scope:
  - Define measurable gate: zero unresolved drift classes tied to YAML for N consecutive checks.
  - Validate no active consumer depends on `CMDB_SOURCE=yaml` path.
  - Produce deletion-execution checklist (separate change window).
- Hard cap: **2h**
- Output: go/no-go gate doc + deletion checklist.

## Sequencing and prerequisites

1. SD-01 (boundary enforcement)
   prerequisite for all others; establishes source-of-truth routes.
2. SD-02 (identity contract)
   prerequisite for reliable drift classification and alert dedupe.
3. SD-03 (automation)
   depends on SD-01 + SD-02 definitions.
4. SD-04 (YAML retirement gate)
   depends on stable SD-03 runs and zero YAML-dependent consumers.

## Backlog vs in-scope split

### In scope (this deliverable family)
- Hybrid authority enforcement.
- Identity/schema contract for overlapping CMDB entities.
- Automated drift detection and reporting.
- YAML fallback deletion gate definition.

### Explicit backlog (out of scope for D-17-45 execution)

- Full runtime-only canonical migration (Option B path).
- Runtime registry MCP surface uplift (D-17-32 Gap X1 sibling), unless required minimally for SD-03 output transport.

## Risk controls

- Read-only doctrine for detection jobs (no CMDB mutation).
- Hash-only verification for any credential-handling checks.
- Surface-back gate retained before any destructive retirement action (YAML deletion execution is separate from this intake scope).
